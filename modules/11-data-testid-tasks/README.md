# Инструкция для Cursor: Jira-задачи на добавление data-testid

## Быстрый старт (копипаст)

```
Открой и следуй: modules/11-data-testid-tasks/README.md

Файл с локаторами: <ссылка на файл>
Создай Jira-задачу на data-testid по локаторам без data-testid из этого файла.
(Опционально: укажи команду, например «команда Chipmunk».)
```

---

## Цель модуля

Пользователь даёт ссылку на файл с локаторами → ты проходишь по файлу, находишь локаторы без data-testid → составляешь список и создаёшь Jira-задачу на разметку.

---

## 1. Файл локаторов и проход по нему

Пользователь даёт **ссылку на файл с локаторами** (например модуль locators или файл с `getByTestId` / `locator` / селекторами).

Ты:
1. Открываешь этот файл (по ссылке или пути).
2. Проходишь по всем локаторам и определяешь, у каких **нет data-testid** (например: используются `getByRole`, `locator('...')`, CSS-класс, текст, XPath и т.п., но не `getByTestId('...')` или аналог с data-testid).
3. По этим локаторам составляешь список для задачи: элемент/назначение локатора и где в коде/компоненте добавить атрибут (файл, компонент — по контексту из файла локаторов). Имена data-testid не предлагать.

---

## 2. Создание Jira-задачи на добавление data-testid

По результатам прохода по файлу локаторов создай Jira-задачу на разметку UI-элементов атрибутами `data-testid` (список из п.1).

### MCP-инструменты

Создание задачи в Jira выполняй через **MCP Atlassian (Jira)**. Перед вызовом проверь схему инструмента (список доступных инструментов и параметров — в дескрипторах MCP, папка `mcps/user-mcp-atlassian/tools/` или аналог в твоём окружении). Обычно это инструмент типа `jira_create_issue`: передаёшь project key (например проект DrillSpot), issue type (Task/Subtask), поля Summary, Description, Labels, Parent (если подзадача) и т.д. по шаблону ниже. Assignee, Fix versions и другие кастомные поля — по правилам проекта или уточни у пользователя.

**Важно при вызове `jira_create_issue`:**
- **Parent** передавать **строкой** ключа задачи, например `"DS-9340"`, а не объектом `{"key": "DS-9340"}` (иначе вызов может завершиться ошибкой). В `additional_fields` задаётся так: `"parent": "DS-9340"`.
- **Story Points** всегда ставить **1**. Поле в Jira: **customfield_10122**. В `additional_fields`: `"customfield_10122": 1`.
- **Команда (Team):** если пользователь при создании задачи указывает команду (например «Chipmunk») — заполнять поле **customfield_10739**, значение — массив строк: `["Chipmunk"]`. В `additional_fields`: `"customfield_10739": ["Chipmunk"]`.

### Шаблон задачи

Пример: `DS-12116` (`https://starsteer.atlassian.net/browse/DS-12116`)

- **Labels:** autotest
- **Parent:** `DS-9340 Drillspot. Autotests.`
- **Story Points:** всегда `1`; в `additional_fields`: `"customfield_10122": 1`
- **Команда:** если пользователь указывает команду (напр. Chipmunk) — в `additional_fields`: `"customfield_10739": ["Chipmunk"]`
- **Summary:** `Drillspot. Autotest. Add test-attribute for UI-elements from <область/экран>`
- **Description** (шаблон):

  ```
  Прилагаю список элементов без data-testid (куда добавить атрибут):
  - <элемент/локатор 1> (компонент/файл, куда добавить)
  - <элемент/локатор 2> ...
  
  Screenshots:
  - <добавить скриншоты при необходимости>
  ```

Требования к оформлению:

- Заполняй **Assignee**, **Labels**, **Parent** (или эпик), **Story Points** (всегда 1, customfield_10122), **Команду** (customfield_10739, если пользователь указал) по правилам проекта.
- Summary — кратко область/экран, для которого добавляются атрибуты.
- В описании — список «элемент/локатор + место в коде» (из прохода по файлу локаторов). Имена data-testid не предлагать.
