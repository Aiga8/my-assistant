# 🧭 Стратегия маппинга автотестов на TestRail (3 этапа)

## 🎯 Цель

Построить воспроизводимый, прозрачный и масштабируемый процесс сопоставления UI-автотестов с тест-кейсами в TestRail, разделив его на три независимых этапа:

1. **Формирование описаний автотестов** → [01_autotest-description-generation.md](01_autotest-description-generation.md)  
2. **Экспорт кейсов из TestRail** → [02_export_testrail_section_subtree.md](02_export_testrail_section_subtree.md)  
3. **Маппинг (сопоставление и фиксация статусов)** → [03_autotest-to-testrail-mapping.md](03_autotest-to-testrail-mapping.md)

Разделение этапов позволяет:
- изолировать источники ошибок,
- минимизировать ручные правки,
- упростить повторное использование данных,
- поддерживать «источник истины» без пересоздания артефактов.

---

## Краткое описание этапов

| Этап | Назначение | Результат |
|------|------------|----------|
| **1** | Стандартизированное описание каждого автотеста (поведение, шаги, проверки) в Markdown | Файл `results/10-autotest-coverage/<задача>_<область>.md` |
| **2** | Полная выгрузка кейсов секции и всех дочерних секций TestRail (API + пагинация) | `testrail-section-<ID>-subtree-cases.{json,md}` |
| **3** | Сопоставление описаний с выгрузкой, присвоение статусов Covered / Partial / Not found | Отчёты по сабсетам + агрегированный `all-tests-testrail-mapping.md` |

Все технические детали, правила и промты — в файлах этапов выше.

---

## Заготовки промптов (копируй и подставь свои данные)

Используй эти формулировки для запуска каждого этапа отдельно.

**Этап 1 — Формирование описаний автотестов:**
```
Выполни формирование описаний автотестов, согласно промту в файле modules/10-autotest-coverage/01_autotest-description-generation.md.
Номер задачи: <>.
Путь к спекам: <tests from solo-rtm>.
```

**Этап 2 — Экспорт кейсов из TestRail:**
```
Выполни экспорт кейсов из TestRail, согласно промту в файле modules/10-autotest-coverage/02_export_testrail_section_subtree.md.
ID секции TestRail — <>.
```

**Этап 3 — Маппинг автотестов на TestRail:**
```
Выполни маппинг автотестов на TestRail, согласно промту в файле modules/10-autotest-coverage/03_autotest-to-testrail-mapping.md.
Файл с описаниями автотестов (результат этапа 1): <>.
Файл с выгрузкой кейсов TestRail (результат этапа 2): <>.
```

---

## Итоговый флоу

1. **Этап 1:** анализ spec-файлов → генерация описаний → сохранение в `results/10-autotest-coverage/`.
2. **Этап 2:** запуск скрипта экспорта по `section_id` → получение `.json` и `.md` выгрузки.
3. **Этап 3:** (при необходимости) фильтрация тестов в сабсет → маппинг на выгрузку → статусы и отчёты.
4. **Агрегация:** пересборка рабочего файла без дублей, сортировка по номеру теста, поддержка точечных фиксов.

---

## Гарантии процесса

- Один автотест = одна запись в рабочем файле.
- Полная трассируемость от автотеста к TestRail-кейсу.
- Возможность точечного обновления без полного пересчёта.
- Независимость этапов: можно обновить выгрузку TestRail или перегенерировать описания по отдельности.
- Повторяемость результата при одинаковых входных данных.
