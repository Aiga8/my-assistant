# Trajectory Spreadsheet — Field Clearing Behavior (Analysis)

Рабочий документ по сценариям очистки полей в Trajectory Spreadsheet (сайдбар). Основан на анализе баг-репортов и **комментариев** к ним (решения разработчиков, дизайнеров, QA).

**Источники:**
- [DS-12189](https://starsteer.atlassian.net/browse/DS-12189) — Spreadsheet. Clear "X" icon does not clear fields; manual clear behaves inconsistently (P0)
- [DS-12488](https://starsteer.atlassian.net/browse/DS-12488) — New Spreadsheets. Impossible to clear Xsrf, Ysrf, Vertical and Horizontal fields in sidebar (P0)
- [DS-12538](https://starsteer.atlassian.net/browse/DS-12538) — New Spreadsheets. Trajectory. Clearing Vertical and Horizontal fields in sidebar causes 400 Bad Request
- [DS-12549](https://starsteer.atlassian.net/browse/DS-12549) — Trajectory. Erasing Horizontal or Vertical uncertainty with keyboard sets input value to empty instead of "0"

**Референсы из комментариев:** аналитика [DS Survey Spreadsheets AN-498](https://starsteer.atlassian.net/wiki/spaces/REQ/pages/4749000735/DS+Survey+Spreadsheets+AN-498#Location-Uncertainty) (Location Uncertainty); автотест **TA-012** (проверка очистки полей сайдбара траектории).

**Переписка (Slack, Ф. Катуров):** ожидаемое поведение для TA-012 и боковой панели траектории скважины — **поля, которые при очистке должны становиться пустыми** (на сервер уходит «нет значения»): Xsrf, Ysrf, Vertical, Horizontal (кнопка «очистить» или стирание значения). **Поля, которые при очистке должны показывать и отправлять 0:** Grid Convergence, Azimuth VS GN, KB Elevation. Общее правило: nullable → пустая строка, non-nullable → 0. Эта формулировка совпадает с первоначальным ОР в DS-12488 и аналитикой; после 400 Bad Request (DS-12538) для Vertical/Horizontal принято текущее поведение: при очистке возврат к 0 (см. раздел 1 и «Итог по Vertical/Horizontal» ниже).

## 1.1. Поля сайдбара Trajectory Spreadsheet (возможность очистки)

Список по структуре сайдбара (Wellbore Trajectory). Для Well Plan и Typewell часть полей read-only или отсутствует (см. аналитику AN-498).

---

### Well

- **Name** — Editable. Required (макс. 200 символов). Очистка: в багах не описана; для текстового поля может быть запрещена или подчиняться общим правилам.
- **API** — Editable. Optional. Очистка: не фигурирует в DS-12189/12488/12538/12549.
- **Operator** — Editable. Optional. Очистка: не фигурирует в багах.

### Surface Location

- **Xsrf** — Editable. Optional. Очистка: **nullable** — при очистке (кнопка «X» или Backspace + blur) поле пустое, на сервер уходит «нет значения», не 0.
- **Ysrf** — Editable. Optional. Очистка: **nullable** — при очистке поле пустое, на сервер уходит «нет значения», не 0.
- **Grid Convergence** — Editable. Optional (допустимо 0). Очистка: **non-nullable** — при очистке отображается и отправляется 0.
- **Azimuth VS (GN)** — Editable. Optional (допустимо 0). Очистка: **non-nullable** — при очистке отображается и отправляется 0.

### Depth Reference

- **Type (Land/Offshore)** — Editable. Required (выбор из двух). Очистка: не применима (выбор значения).
- **KB Elevation** — Editable. Optional (допустимо 0). Очистка: **non-nullable** — при очистке отображается и отправляется 0 (TA-012, DS-12549).
- **Air Gap** — Read-only (расчётное). Очистка недоступна.
- **Ground Level** — Editable при Type = Land. Optional. Очистка: не фигурирует в багах; уточнять по аналитике (0 vs пустое).
- **Water Depth** — Editable при Type = Offshore. Optional. Очистка: не фигурирует в багах.

### Location Uncertainty

- **Vertical** — Editable. Optional (диапазон 0–100 000). Очистка: **non-nullable** — при очистке поле возвращается к 0, на сервер уходит 0 (не пустая строка); иначе возможен 400 (DS-12538, DS-12549).
- **Horizontal** — Editable. Optional (диапазон 0–100 000). Очистка: **non-nullable** — при очистке поле возвращается к 0, на сервер уходит 0.

### Magnetics

- **Sample Date** — Editable. Обязательность зависит от модели. Очистка: в багах не описана; дата — отдельная логика (календарь, сброс).
- **Model Name** — Editable. Required (выбор модели). Очистка: выбор из списка; сброс/очистка в багах не описаны.
- **Field Strength** — Editable при User Defined / read-only при расчётной модели. Очистка: не фигурирует в багах.
- **Dip Angle** — Editable при User Defined / read-only при расчётной модели. Очистка: не фигурирует в багах.
- **Declination** — Editable при User Defined / read-only при расчётной модели. Очистка: не фигурирует в багах.

### Survey Program

- **Start MD** — Read-only (вычисляется). Очистка недоступна.
- **Stop MD** — Editable. Required в строке таблицы. Очистка ячейки — отдельный контекст (валидация Stop MD > Start MD), в багах сайдбара не описана.
- **Toolcode** — Editable. Optional. Очистка: в багах сайдбара не фигурирует.

---

**Примечания:**
- Для **Well Plan Trajectory** часть полей (в т.ч. Depth Reference от родителя, Location Uncertainty, Survey Program) read-only; поведение очистки относится к редактируемым полям Wellbore/Typewell.
- **Time Zone** и **Spud Date** (DS-10818) в сайдбаре S1 Wellbore — редактируемые; в багах DS-12189/12488/12538/12549 не упоминаются, поведение при очистке уточнять по аналитике.
- Поля с явно заданным в багах поведением при очистке: **Xsrf, Ysrf** → пустое; **Grid Convergence, Azimuth VS GN, KB Elevation, Vertical, Horizontal** → 0.

## 2. Expected behavior when a value is cleared

### 2.1 Clear control (“X” icon)

- **Ожидание:** Кнопка «X» (крестик) очищает значение так же, как ручное удаление (Backspace).
- **Поведение по полям:**
  - **Nullable (Xsrf, Ysrf):** поле отображается пустым; на сервер уходит пустое значение (null / отсутствие значения), не 0.
  - **Non-nullable (Vertical, Horizontal, KB Elevation, Azimuth VS GN, Grid Convergence):** после очистки поле отображает и отправляет **0** (ноль — допустимое значение).

### 2.2 Manual clear (Backspace / Delete)

- **Ожидание:** Ручная очистка до пустой строки и уход фокуса дают тот же результат, что и очистка через «X».
- **Nullable:** пустое отображение и пустое значение на сервере.
- **Non-nullable:** отображение и значение 0; в PATCH уходит 0, а не предыдущее значение и не «пустая строка» как число.

### 2.3 Persistence after clear

- После очистки и сохранения при переключении вкладки и возврате значение не должно «возвращаться» к старому: отображается либо пустое (nullable), либо 0 (non-nullable) в зависимости от типа поля (DS-12189 Case 1).

### 2.4 Non-nullable: revert to 0 when cleared

- Для полей с типом non-nullable (в т.ч. Vertical, Horizontal): если пользователь стирает значение клавиатурой до пустого и уводит фокус (Tab / клик вне поля), поле должно вернуться к **0**, а не оставаться пустым (DS-12549).

## 3. Validation rules

- **Nullable поля (Xsrf, Ysrf):** пустое значение допустимо; при очистке на сервер передаётся «нет значения» (null/empty/undefined), не 0.
- **Location Uncertainty (Vertical, Horizontal):** диапазон 0–100 000; при очистке допустимое значение — **0** (non-nullable). Пустое значение не принимается: при ручной очистке до пустого и blur поле должно вернуться к 0; в теле запроса передаётся **0**, не пустая строка (иначе сервер может вернуть 400 Bad Request — DS-12538).
- **Non-nullable поля (KB Elevation, Azimuth VS GN, Grid Convergence):** при очистке допустимое значение — 0; в UI и в запросе должно быть 0, не пустое.
- **PATCH после очистки:** в теле запроса должно уходить актуальное состояние (пустое/null для nullable, **число 0** для non-nullable), а не предыдущее значение (DS-12189 Case 3) и не пустая строка для числовых non-nullable полей.
- **Ограничение (из комментариев DS-12538, DS-12549):** для non-nullable полей при сохранении после очистки в теле запроса передаётся значение **0**, а не пустая строка — иначе возможна ошибка 400 или некорректное отображение после перезагрузки.

### Реализация (из комментария DS-12549)

- **Логика отображения:** если приходит `value` (число, в т.ч. 0), а пользователь стёр текст до пустого — показывать значение из prop (0). Если `value` — `undefined` — показывать пустое поле. Устраняет баг, когда при внешнем 0 и пустом инпуте компонент не обновлял отображение.
- **Регрессия:** в комментарии DS-12549 перечислены SectionsAndRuns, Spreadsheet (ячейки), Rigs, Ribbon, Widgets, Settings — для числовых non-nullable инпутов ожидается сброс к 0 при стирании до пустого и blur.

### 5.1. Тесты, которые уже затрагивают очистку

- **TR#183719** (section 8193) — *Clear the date in the Sample date*. Единственный кейс с явной «очисткой» в названии. Шаги: в секции Magnetics в календаре Sample Date очистить дату и нажать Apply. ### 5.3. Поля, по которым нет покрытия в выбранных секциях

- **Well (6981):** Name, API, Operator — нет ни одного кейса, проверяющего **очистку** (пустое значение, кнопка «X», сохранение пустого).
- **Surface Location (7796):** Xsrf, Ysrf, Grid Convergence, Azimuth VS (GN) — нет кейсов про **очистку** и ожидаемый результат (пустое для Xsrf/Ysrf, 0 для GC и Azimuth VS GN).
- **Depth Reference (7816):** KB Elevation, Ground Level, Water Depth — нет кейсов про **очистку** и отображение/отправку 0 после очистки.
- **Location Uncertainty (8038):** Vertical, Horizontal — нет кейсов про **очистку** и возврат к 0 (или пустое).
- **Magnetics (8193):** Field Strength, Dip Angle, Declination — нет кейсов про **очистку** (кнопка «X» или стирание + blur) и ожидаемое значение после очистки.
- **Survey Program (8152):** Stop MD, Toolcode — очистка ячейки/поля в контексте «пустое vs 0» или сброс выбора в выгрузке не покрыта как отдельный сценарий.

**Итог:** сценарии очистки полей сайдбара (кнопка «X», Backspace + blur, пустое vs 0 для перечисленных полей) в выгрузке TestRail по секциям 6980, 6981, 7796, 7816, 8038, 8193, 8152 **не покрыты**. Покрытие даёт автотест TA-012 (вне этой выгрузки). Существующие кейсы проверяют ввод, валидацию и отправку изменённых атрибутов, но не явное поведение при очистке значения.
