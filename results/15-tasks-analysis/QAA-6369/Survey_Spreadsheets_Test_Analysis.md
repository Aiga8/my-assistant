## 1. Базовая функциональность Survey Spreadsheets

*Согласно основной аналитике AN-498 (актуальна, новые фичи её не отменяют).*

### Назначение
Отображение и редактирование Spreadsheets для **Well** и **Well Plan** в DrillSpot: работа с траекторией скважины и связанными параметрами без перехода в Starsteer.

### Переход в Spreadsheet
- **ПКМ** по объекту (Well, Typewell, Well plan и т.д.) в дереве → пункт **Spreadsheet**
- Кнопка **Well Trajectory** в рибоне велла  

В обоих случаях открывается сервисная таба **Well Trajectory** и выполняется редирект на неё.

### Структура табы Well Trajectory (порядок блоков)
1. **Well** — Name, API, Operator (текст, макс. 200 символов)
2. **Surface Location** — Xsrf, Ysrf, Grid Convergence, Azimuth VS (GN); диапазоны и точность отображения (1/2/5/6 знаков в зависимости от поля)
3. **Depth Reference** — Type (Land/Offshore), KB Elevation, Air Gap (расчётное), Ground Level / Water Depth
4. **Location Uncertainty** — Vertical, Horizontal (0–100 000)
5. **Magnetics** — Sample Date, Model Name (User Defined / WMM2025 / IGRF14), Field Strength, Dip Angle, Declination; расчётные поля при выбранной модели; валидация даты и модели
6. **Survey Program** — таблица Start MD / Stop MD / Toolcode, кнопки добавления/удаления строк; Stop MD > Start MD; сортировка тулкодов по prefix
7. **Таблица траектории** — MD, INCL, AZIM TN, AZIM GN, TVD, N/S (G/N), E/W (G/N), X (G/N), Y (G/N), TVDSS, VS, DLS; строка tie-in редактируемая; расчёты через Geomath; правила по Grid Convergence (когда AZIM TN / AZIM GN редактируемые или расчётные)

### Well Plan Trajectory
- Отдельный набор блоков: Well Plan (Name, API, Operator), Surface Location, Depth Reference (часть полей read-only от родительского велла), Location Uncertainty (read-only), Survey Program (таблица read-only, данные родительского велла)
- Плейсхолдеры «as parent well» для Xsrf, Ysrf, KB Elevation при пустых значениях
- При выборе в дереве другого велла отображается Trajectory выбранного велла

### Lock
- Таба открывается и при заблокированной скважине; все поля только для просмотра.

### Редактирование таблицы траектории
- **Delete rows** — одна или несколько строк; при выделении всей таблицы/столбца удаление недоступно
- **Insert row** — пустая строка ниже выбранной; подсветка обязательных полей (MD, INCL, AZIM GN или AZIM TN в зависимости от Grid Convergence) до заполнения всех трёх
- **Insert copied cells** — вставка из буфера; невалидные значения не вставляются
- **Copy** / **Copy with headings** / **Paste** — с учётом редактируемых полей; при выделении строк/столбцов — Action Bar
- Невалидные строки не сохраняются; при редактировании существующей строки в невалидное состояние — откат к последнему валидному при обновлении/повторном открытии

### Ключевые пользовательские сценарии (из аналитики)
- Открытие Well Trajectory через дерево (ПКМ → Spreadsheet)
- Добавление новой строки (подсветка полей, пересчёт)
- Удаление строки (пересчёт оставшихся)
- Копирование с заголовками (Copy with headings)
- Вставка скопированных данных извне (Paste с учётом ограничений и пересчёта)

## 2. Изменения из DS-10818 — New Spreadsheets

*Согласно [AN New Spreadsheets DS-10818](https://starsteer.atlassian.net/wiki/spaces/RTM/pages/5311594500/AN+New+Spreadsheets+DS-10818). Фича вводит единую систему спредшитов; ниже — только то, что касается Survey Spreadsheets (Trajectory: Wellbore, WellPlan, Typewell).*

### Кратко: что изменилось

| Область | Изменение |
|--------|-----------|
| **Столбцы S1 (Wellbore Trajectory)** | Добавлены столбцы **эллипсов ошибок (EoU):** High-side Error, Lateral Error, Vertical Error, Along-hole Error, Semi-major Error, Semi-minor Error, Semi-minor AZ GN. Добавлены **Distance to Well Plan:** Vertical (above/below), Horizontal (left/right), Distance (пустые, если не задан starred wellplan). |
| **Столбцы S14 (WellPlan), S20 (Typewell)** | Добавлены только столбцы EoU. Столбцов Distance to Well Plan нет. |
| **Поля из Ribbon** | **Time Zone** и **Spud Date** перенесены в спредшит **S1. Wellbore - Trajectory** и удалены из Ribbon Well. В S14 и S20 этих полей нет. |
| **Настройка столбцов** | Порядок, видимость и ширина столбцов настраиваются и **сохраняются по типу спредшита** (в user settings). При открытии любого другого спредшита того же типа (другая скважина) настройки применяются. Есть кнопка **«Сбросить к по умолчанию»**. |
| **Ограничение настройки** | Столбец **MD** в S1 нельзя скрыть. Остальные (в т.ч. новые EoU, Distance to Plan) можно скрывать и менять порядок. |
| **Lock Object** | **Исключение для Trajectory (S1):** при заблокированной скважине редактирование в спредшите **недоступно**. Для остальных спредшитов Lock на редактирование не влияет. Для S14/S20 редактирование по-прежнему возможно (Lock на них не распространяется). |
| **Копирование** | При копировании (Ctrl+C или контекстное меню) в буфер попадают значения **без округления** (полная точность). Поддержка копирования столбцов с заголовками. Контекстное меню браузера отключено, показывается кастомное с «Copy». |
| **Единая точка входа** | Открытие только через пункт «Spreadsheet» в контекстном меню объекта. В заголовке окна — наименование и источник данных (datasource). |
| **Юниты** | Юниты по настройкам проекта; пользователь не может менять юниты в столбцах. |
| **Сессионные фильтры** | Фильтры по столбцам доступны; **не сохраняются** — после закрытия спредшита сбрасываются. |
| **Стриминг** | Данные в открытом спредшите обновляются при поступлении новых данных; таблица **автоматически прокручивается** к новой последней строке. |
| **Сайдбар** | Сайдбар можно скрывать/показывать; настройка **не сохраняется** (при новом открытии снова виден). |

### Сценарии, которые могли измениться

- **Открытие Trajectory** — тот же пункт «Spreadsheet», но заголовок окна должен содержать наименование и datasource; в S1 в интерфейсе спредшита отображаются Time Zone и Spud Date (в Ribbon их больше нет).
- **Редактирование при Lock** — для Wellbore Trajectory (S1) при Lock Object редактирование запрещено; для WellPlan (S14) и Typewell (S20) — по-прежнему разрешено.
- **Копирование** — ожидание «полная точность при копировании», контекстное меню DrillSpot вместо браузерного, копирование столбцов с заголовками.
- **Отображение таблицы** — появление новых столбцов (EoU; для Wellbore ещё Distance to Well Plan при наличии starred wellplan); столбец MD всегда видим, остальные настраиваемые.
- **Настройка столбцов** — сохранение порядка/видимости/ширины между сессиями для данного типа спредшита; сброс к по умолчанию.
- **Фильтры** — работа только в рамках сессии, после закрытия спредшита фильтры сбрасываются.
- **Появление новых данных** — автообновление таблицы и автоскролл к последней строке (стриминг).

### Новые сценарии (появились с DS-10818)

- **Настройка и сохранение вида столбцов:** изменить порядок/ширину/видимость столбцов в S1 → закрыть → открыть другой Wellbore Trajectory → настройки те же; сброс к по умолчанию восстанавливает дефолтный вид и сохраняет его.
- **Копирование с полной точностью:** в ячейке отображается округлённое значение → Ctrl+C → в буфере неокруглённое значение; копирование диапазона/столбцов с заголовками в формате для Excel (табы/переносы строк).
- **Единая точка входа и заголовок:** ПКМ по Trajectory → «Spreadsheet» → в заголовке окна отображаются наименование и источник данных.
- **Юниты проекта:** открыть Trajectory в метрическом/футовом проекте — заголовки столбцов и значения в соответствующих юнитах; смена юнитов в спредшите недоступна.
- **Сессионные фильтры:** применить фильтр по столбцу → закрыть спредшит → открыть снова → фильтр сброшен.
- **Стриминг:** спредшит Trajectory открыт → в объект поступает новая точка траектории → новая строка появляется в таблице без перезагрузки, таблица прокручивается к ней.
- **Новые столбцы S1:** наличие столбцов EoU и Distance to Well Plan (для Wellbore); при отсутствии starred wellplan столбцы Distance to Well Plan пустые.
- **Новые столбцы S14/S20:** наличие столбцов EoU; отсутствие Distance to Well Plan и полей Time Zone/Spud Date.
- **Фиксация MD:** в настройках столбцов S1 столбец MD нельзя скрыть.
- **Lock только для S1:** Wellbore заблокирован → в S1 редактирование недоступно; в S14/S20 при Lock (родительского велла) редактирование доступно.

## 3. Изменения из DS-9914 — Tabs in Multiple Windows

*Согласно [AN Tabs in Multiple Windows DS-9914](https://starsteer.atlassian.net/wiki/spaces/RTM/pages/5281710129/AN+Tabs+in+Multiple+Windows+DS-9914). Ниже — только то, что касается вкладок, нескольких окон и поведения Spreadsheets (в т.ч. Trajectory).*

### Работа вкладок

| Аспект | Поведение |
|--------|-----------|
| **Несколько экземпляров Trajectory Spreadsheet** | Trajectory Spreadsheet (как Section and Runs, Rigs) можно открывать в нескольких экземплярах; каждое открытие создаёт **новую вкладку** в Workspace. |
| **Повторное открытие того же Wellbore** | При открытии спредшита для **уже открытого** Wellbore новая вкладка **не создаётся** — фокус переключается на существующую вкладку с этим спредшитом и Wellbore. |
| **Ручное совпадение DataSource** | Допускается вручную менять DataSource у нескольких открытых спредшитов так, чтобы они показывали один и тот же Wellbore (в будущем может быть ограничено). |
| **Заголовок вкладки Trajectory Spreadsheet** | Wellbore: `[Иконка Wellbore] Wellbore Trajectory [Имя Wellbore]`. Typewell: `[Иконка Typewell] Typewell Trajectory [Имя Typewell]`. Имя Wellbore/Typewell должно быть хорошо читаемым (в т.ч. на больших экранах). |
| **Активная вкладка** | В любой момент активна только одна вкладка. **Дерево объектов** и **URL** всегда соответствуют активной вкладке (её DataSource). |
| **Смена DataSource вкладки** | Чтобы сменить DataSource у вкладки со спредшитом: сделать вкладку активной (клик по ней), затем выбрать другой Wellbore в дереве. Меняется только эта вкладка. |
| **Равенство вкладок** | Окна (дашборды), Section and Runs, Trajectory Spreadsheet, Rigs — все обрабатываются как вкладки: общее Workspace, перетаскивание, группировка, изменение размеров, Pop-out. |
| **Сохранение Workspace** | Набор открытых вкладок, их порядок, размеры групп, DataSource каждой вкладки сохраняются автоматически и восстанавливаются при новой сессии / перезагрузке страницы. |

### Работа нескольких окон (Pop-out)

| Аспект | Поведение |
|--------|-----------|
| **Открепление вкладки** | Любую вкладку, в т.ч. Trajectory Spreadsheet, можно открепить (Pop-out) в отдельное окно браузера (иконка в заголовке вкладки; в десктопе — также перетаскивание за пределы окна). |
| **Pop-out как Workspace** | Откреплённое окно — полноценное мини-Workspace: можно добавлять вкладки, перетаскивать вкладки между основным и pop-out окном, менять компоновку. |
| **Единое дерево** | Дерево объектов одно на всё приложение (в основном окне). Им задаётся DataSource для **всех** вкладок — и в основном, и в pop-out окнах. |
| **Закрытие pop-out** | При закрытии pop-out окна все его вкладки возвращаются в основное окно. При перезагрузке страницы все вкладки оказываются в одном окне. Положение/размер pop-out окон ОС не сохраняются. |
| **Redock** | Функция «Redock» закрывает все откреплённые окна и возвращает их вкладки в основное окно (аналогично StarSteer). |
| **Перетаскивание между окнами** | Вкладки можно перетаскивать между основным и pop-out окнами (и между несколькими pop-out). |
| **URL в pop-out** | У каждого pop-out окна своя адресная строка; URL строится по тем же правилам (активная вкладка в этом окне). В основном окне при откреплении вкладки URL обновляется под новую активную вкладку. Все окна работают в контексте **одного и того же проекта**; смена проекта в любом окне меняет проект во всём Workspace. |

### Поведение Spreadsheets (Trajectory в частности)

| Аспект | Поведение |
|--------|-----------|
| **DataSource при открытии** | DataSource спредшита задаётся объектом, из которого его открыли (выбранный Wellbore/Typewell в дереве). |
| **Смена проекта (NEW 17.10.2025)** | При переключении проекта открытые спредшиты **не закрываются**. У каждого спредшита DataSource автоматически переключается на «наиболее актуальный» wellbore в новом проекте (логика выбора wellbore как для зон). Если открыто несколько экземпляров одного типа (напр. три Trajectory Spreadsheet), в первом — самый актуальный wellbore, в остальных — следующие по актуальности. Rigs не меняются. |
| **Кнопки открытия Spreadsheet** | Кнопки открытия Spreadsheets (в Ribbon и др.) не блокируются и не скрываются, даже если такая вкладка уже открыта. |
| **URL и шаринг** | URL отражает активную вкладку (ID окна/таба, проект, Wellbore). Можно делиться ссылкой на конкретный вид (Trajectory + wellbore). При переходе по ссылке: если такая вкладка уже открыта — переход на неё и обновление DataSource из ссылки; если нет — открытие в новой вкладке с DataSource из ссылки. При смене проекта по URL все вкладки (в т.ч. спредшиты) перестраивают DataSource под новый проект. |

### Влияние на тесты Survey Spreadsheets

- **Вкладки:** проверки «открыть Trajectory → одна сервисная таба» дополняются сценариями «несколько вкладок Trajectory с разными Wellbore», «повторное открытие того же Wellbore → фокус на существующей вкладке», «заголовок вкладки с именем Wellbore/Typewell», «смена DataSource через выбор другого Wellbore в дереве при активной вкладке спредшита».
- **Несколько окон:** сценарии «Pop-out вкладки Trajectory в отдельное окно», «дерево в основном окне управляет DataSource pop-out», «закрытие pop-out → вкладки возвращаются в основное окно», «перетаскивание вкладки Trajectory между основным и pop-out окном».
- **Сохранение и восстановление:** «компоновка с несколькими Trajectory вкладками и разными Wellbore сохраняется и восстанавливается после перезагрузки».
- **Смена проекта:** «при смене проекта открытые Trajectory Spreadsheet не закрываются; DataSource переключается на актуальные wellbore нового проекта (распределение по экземплярам)».
- **URL:** «URL отражает активную вкладку (в т.ч. Trajectory + Wellbore)», «открытие по ссылке добавляет/обновляет вкладку с нужным DataSource».
