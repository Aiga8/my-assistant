## Секция 31067 — Window tabs in new layout

### Тест 357577 — "Verify window name display in tab from window list"
- **Консистентность:** В expected: «В **табе** отображается» → для единообразия с остальными кейсами секции лучше «Во **вкладке** отображается».
- Логика кейса корректна.

### Тест 357578 — "Verify public window icons in tabs"
- **Стилистика:** «перед наименованием» — в других кейсах используется «имя окна»; при желании унифицировать: «перед **названием/именем окна**».
- Замечаний по шагам и ожиданию нет.

### Тест 357593 — "Displaying the window name in focus and without focus"
- **Грамматика (expected шага 2):** «Публичное окно из **шага один**» → «из **шага 1**» или «из **первого шага**» («шага один» звучит неестественно).
- **Грамматика (шаг 2):** «Открыть новое окно публичное или приватное» → «Открыть новое **публичное или приватное** окно» (порядок слов).
- Ожидания по формату заголовка (иконка + имя окна ± веллбор) соответствуют аналитике 1FR1.3-R3.

### Тест 357579 — "Verify wellbore name display in window tab"
- **Грамматика (шаг):** «поочередно» → «**поочерёдно**» (буква ё).
- Ожидание сформулировано ясно.

### Тест 357594 — "Displaying the window name without binding to a wellbore"
- **Грамматика (шаг):** «Проверить отображение **отсутствие** имени веллбора» → «Проверить **отсутствие отображения** имени веллбора» или «Проверить, **что** в заголовке **не отображается** имя веллбора».
- **Терминология (preconditions):** «в проекте отсутствуют веллы» — при необходимости уточнить «**веллборы**» или «скважины» для единообразия с остальными кейсами.
- Ожидание корректно.

### Тест 357580 — "Verify wellbore selection synchronization between tree and widgets"
- В конце expected можно добавить точку.
- Замечаний по смыслу нет.

### Тест 357581 — "Verify Add Zone buttons removal from Dashboard ribbon"
- **Соответствие названию и аналитике:** Название кейса — проверка **удаления** кнопок Add Zone (removal). В ожидании перечислено, что **отображается** в риббоне, в том числе «контрол **Add Analytics Zone**». По 1FR1.2-R2 кнопки «Add Monitoring Zone» и «Add Analytic Zone» должны быть удалены из интерфейса; для Пемекса допускается оставить Add Analytic Window. Нужно уточнить: если кнопки зон действительно убраны — в ожидании не должно быть «Add Analytics Zone» (или указать исключение, например в preconditions). Если по дизайну кнопка остаётся — скорректировать название кейса (например, «Verify Dashboard ribbon controls» или «Verify Add Zone buttons removal (Add Analytics excluded for Pemex)»).
- **Структура expected:** В списке последний пункт оформлен как «<li><p>контрол Publish Window</p></li>» — для единообрадия лучше «<li>контрол Publish Window</li>» (без вложенного <p>).

## Рекомендации

### Шаблонные правки
- **«табе»** → **«вкладке»** (357577) для единообразия.
- **«поочередно»** → **«поочерёдно»** (357579).
- Исправить формулировку шага в 357594: «отображение отсутствие» → «отсутствие отображения» или «что не отображается».
- В 357593: «шага один» → «шага 1» / «первого шага»; «окно публичное или приватное» → «публичное или приватное окно».
