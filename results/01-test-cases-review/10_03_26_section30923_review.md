## Секция 30923 — Maximizing the widget

### Тест 356519 — "Checking the tree context menu when the widget is maximized"
- **Preconditions:** «Закрепить развернутое дерево объектов», «Максимизировать любой виджет на дашборде» — формулировки понятны.
- **Шаг / ожидание:** Контекстное меню (ПКМ) на элементе дерева — меню отображается полностью, не перекрыто виджетом, все пункты доступны. Логика корректна.
- **Замечаний нет.**

### Тест 356520 — "Checking the functionality of a collapsed tree when the widget is maximized"
- **Грамматика (preconditions):** «Открыть дашборд добавить» → «Открыть дашборд, добавить» (нужна запятая между действиями).
- **Грамматика:** «Развернуть виджет на все окно» → «Развернуть виджет на всё окно» (в значении «на целое окно» корректно «всё»).
- **Структура (шаг):** Два действия в одном шаге («Свернуть дерево…» и «Попытаться вызвать…») — для однозначности лучше разделить на два шага или явно перечислить: «1) Свернуть дерево объектов (кнопка сворачивания). 2) Вызвать дерево объектов повторно».
- Ожидания сформулированы ясно.

### Тест 356521 — "Checking the ribbon's functionality when the widget is maximized"
- **Грамматика (preconditions):** «Открыть дашборд добавить» → «Открыть дашборд, добавить».
- **Грамматика:** «Развернуть виджет на все окно» → «Развернуть виджет на всё окно».
- **Грамматика (шаг):** В перечне типов риббона указано «трейса» → «трейс» (в списке «дашборд, велл, калькулятор, виджет, трейс» — название типа риббона в одной форме).
- Ожидания корректны.

## Рекомендации

### Шаблонные правки
- **«Открыть дашборд добавить»** → **«Открыть дашборд, добавить»** в preconditions кейсов 356520 и 356521.
- **«на все окно»** → **«на всё окно»** в обоих кейсах (356520, 356521).
- **«трейса»** → **«трейс»** в шаге кейса 356521 (перечень типов риббонов).

### Покрытие

Для секции «Maximizing the widget» этого достаточно; при необходимости можно добавить кейс на закрытие полноэкранного режима (соответствует секции 4027).
