**Summary**

Spreadsheet. Common. Always auto-scroll to newly created row after clicking Add (to keep new row visible)

**Description**

*Context / Links:*
* Область: Spreadsheet. Common
* Related Jira: DS-11861 (Mud Reports. No auto-scroll to newly created report after clicking Add Report button)
* Контекст: обсуждение с дизайнером (решение — always-scroll)

*Description:*
Сейчас в длинных спредшитах/таблицах (когда есть вертикальный скролл) после нажатия кнопки добавления строки (например, {{Add Report}}):
* если пользователь находится не внизу списка и нажимает {{Add ...}}, новая строка создаётся в конце списка, но остаётся *за пределами видимой области* — пользователь не видит результат действия и не может сразу заполнить созданную строку
* если пользователь предварительно проскроллил ближе к низу, автоскролл к новой строке срабатывает не всегда, из‑за чего поведение не единообразно

**Proposed Solution:**
Сделать поведение единым для Spreadsheet.Common: после клика по кнопке добавления строки всегда выполнять автоскролл так, чтобы *созданная строка была видна* (и, по возможности, готова к заполнению).

**Expected Behavior:**
После нажатия {{Add ...}}:
* новая строка добавляется (как сейчас) *в конец списка*
* таблица автоматически прокручивается так, чтобы созданная строка оказалась в видимой области (независимо от текущей позиции вертикального скролла)
* (желательно) фокус устанавливается в первую редактируемую ячейку новой строки

*Notes / Constraints:*
* Решение от дизайнера в обсуждении: “Давайте всегда скроллить к новой строке. Её же ещё заполнить надо, а не просто создать и забыть”.
* Если улучшение должно применяться ко всем новым/существующим спредшитам, рекомендуется реализовывать на уровне общего компонента/поведения Spreadsheet.Common (а не точечно для Mud Reports).
